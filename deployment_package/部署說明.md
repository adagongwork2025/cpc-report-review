# 中油偵測報告系統 - 部署說明文件

## 系統概述

本系統為中油工安監控平台的違規偵測審核與報告系統，整合 iSeek AI 影像辨識技術，提供高處作業和局限空間的違規審核與通報功能。

## 部署需求

### 1. 伺服器環境
- **Web Server**: Nginx / Apache / 任何靜態檔案伺服器
- **支援 HTTPS**: 必須啟用（API 呼叫需要）
- **瀏覽器**: Chrome 90+, Safari 14+, Firefox 88+

### 2. 網路需求
- 需要能存取以下 API 端點：
  - `https://apigatewayiseek.intemotech.com/vision_logic/video?key=`
  - `https://apigatewayiseek.intemotech.com/vision_logic/logs/notify/image?key=`
  - `https://apigatewayiseek.intemotech.com/vision_logic/logs/notify/filter`

### 3. 儲存空間
- 目前資料大小：約 3.3 MB
- 建議預留：至少 1 GB（未來擴充）

## 部署步驟

### 方法一：靜態網站部署（推薦）

1. **上傳檔案**
   ```bash
   # 將整個 deployment_package 目錄上傳到伺服器
   scp -r deployment_package/ user@server:/var/www/cpc-detection/
   ```

2. **設定 Nginx**
   ```nginx
   server {
       listen 80;
       server_name cpc-detection.example.com;

       root /var/www/cpc-detection/deployment_package;
       index index.html;

       location / {
           try_files $uri $uri/ =404;
       }

       # 啟用 CORS（如果需要）
       add_header Access-Control-Allow-Origin *;
   }
   ```

3. **啟用 HTTPS**
   ```bash
   sudo certbot --nginx -d cpc-detection.example.com
   ```

### 方法二：雲端儲存空間（AWS S3 / GCS / Azure Blob）

1. **上傳到 S3 Bucket**
   ```bash
   aws s3 sync deployment_package/ s3://your-bucket-name/ --acl public-read
   ```

2. **啟用靜態網站託管**
   - 在 S3 設定中啟用「靜態網站託管」
   - 設定 index.html 為預設文件

3. **設定 CORS**
   ```json
   [
     {
       "AllowedHeaders": ["*"],
       "AllowedMethods": ["GET"],
       "AllowedOrigins": ["*"],
       "ExposeHeaders": []
     }
   ]
   ```

## 系統運作原理

### 1. 主頁面（index.html）
- 讀取 `REPORT_DATA` 陣列，顯示所有日期的報告列表
- 使用 `localStorage` 記錄是否已完成審核
- 未完成審核前，報告按鈕會顯示為灰色且無法點擊

### 2. 審核頁面（*_審核.html）
- 顯示所有違規記錄的卡片介面
- 可播放影片、查看縮圖
- 提供「違規」、「誤報」按鈕進行審核
- 按下「產生偵測報告」後會：
  - 將審核結果存入 `localStorage`
  - 跳轉到報告頁面

### 3. 報告頁面（*_報告_管理版.html）
- 從 `localStorage` 讀取審核結果
- 僅顯示標記為「違規」的記錄
- 提供統計資訊和匯出功能

### 4. 資料流程
```
API 資料 → 審核工具頁面 → 使用者審核 → localStorage 儲存
                                          ↓
                                      報告頁面讀取 → 顯示結果
```

## 維護指南

### 新增每日資料

**選項 A：使用 Python 腳本自動產生（推薦）**

```bash
# 安裝需求
pip3 install requests

# 執行腳本產生審核工具
cd scripts/
python3 generate_daily_review.py

# 腳本會自動：
# 1. 從 API 抓取資料
# 2. 產生審核頁面和報告頁面
# 3. 更新 index.html 的 REPORT_DATA 陣列
```

**選項 B：手動新增**

1. 在 `2026/04/08/` 建立新目錄（依日期調整）
2. 複製類似的審核和報告 HTML 檔案
3. 修改 DATA 陣列中的資料
4. 更新 `index.html` 的 `REPORT_DATA` 陣列：

```javascript
const REPORT_DATA = [
  // ... 現有資料
  {
    "date": "2026-04-08",
    "category": "高處作業",
    "reviewPath": "2026/04/08/高處作業_審核.html",
    "reportPath": "2026/04/08/高處作業_報告_管理版.html"
  }
];
```

### 清除審核記錄

如需重新審核，可在瀏覽器 Console 執行：

```javascript
// 清除特定日期的審核記錄
localStorage.removeItem('reportData_2026-04-07_高處作業');

// 清除所有審核記錄
for (let key in localStorage) {
  if (key.startsWith('reportData_')) {
    localStorage.removeItem(key);
  }
}
```

## 疑難排解

### 問題 1：影片無法播放

**原因**：HTTPS 要求，或 API 端點無法存取

**解決方式**：
- 確認網站使用 HTTPS
- 檢查防火牆是否允許存取 `apigatewayiseek.intemotech.com`
- 在瀏覽器 Console 查看錯誤訊息

### 問題 2：報告按鈕無法點擊

**原因**：尚未完成審核

**解決方式**：
- 點擊「審核」連結進入審核頁面
- 完成所有記錄的審核（違規/誤報）
- 按下「產生偵測報告」按鈕

### 問題 3：縮圖無法顯示

**原因**：圖片路徑錯誤或 API 無法存取

**解決方式**：
- 檢查圖片 URL 是否正確
- 確認 `image_url` 格式為：
  `https://apigatewayiseek.intemotech.com/vision_logic/logs/notify/image?key={key}`

## API 端點說明

### 1. 取得偵測記錄
```
POST https://apigatewayiseek.intemotech.com/vision_logic/logs/notify/filter

Request Body:
{
  "start_time": "2026-04-07T00:00:00",
  "end_time": "2026-04-07T23:59:59",
  "camera_id": "758",  // 高處作業
  "per_page": 1000,
  "page": 1
}
```

### 2. 取得影片
```
GET https://apigatewayiseek.intemotech.com/vision_logic/video?key={path}

範例：
key=/app/image/kafka_notify_videos/production/20260407/758_高處_142328094.mp4
```

### 3. 取得圖片
```
GET https://apigatewayiseek.intemotech.com/vision_logic/logs/notify/image?key={key}

範例：
key=/app/image/kafka_notify_images/production/20260407/1921685396_120_0_758_53033_142328084.jpg
```
